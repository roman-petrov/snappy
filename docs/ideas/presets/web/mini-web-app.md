# 🧩 Mini web app

## 💡 Concept

**Agent skill.** The user describes a small web application in plain language; the agent, guided by skills, builds it as
a self-contained web app—layout, styling, and light client-side logic. While the app is being created, the user sees a
**live preview** that updates as pages and interactions appear. When ready, Snappy **publishes** the app immediately to
the web: a **link on the site** opens it in any browser, with no separate deploy or hosting step for the user.

For calculators, trackers, mini-games, one-page tools, landing micro-sites, demos, and other one-off apps that are not a
chat transcript, a PDF, or a fixed template. Polished formats such as **greeting pages**, **presentations**, and
**interactive courses** can ship as dedicated skills on the same publish path.

## 👤 How the user experiences it

1. The user starts a skill or asks to build a mini web app.
2. The agent asks step by step: what the app should do, who will use it, main screens or flows, tone, colors, and
   whether it needs forms, charts, media, or saved state in the browser.
3. The user describes requirements, uploads assets or sample data, or points at a reference. The agent proposes a short
   plan; the user confirms or edits it.
4. During generation, the chat runs alongside a **live preview**. The user can click through the app as it takes shape
   and ask for tweaks (“bigger buttons”, “add a second step”, “match this palette”).
5. When satisfied, the user **publishes**; the app is **live on the web** right away and the user receives a **shareable
   link**.
6. Visitors open the link on phone, tablet, or desktop and use the app in the browser—no Snappy account or install
   required unless the author chose sign-in or other gated behavior.
