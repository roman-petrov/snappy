# 🖥️ Interactive presentation page

## 💡 Concept

**Agent skill.** The agent interviews the user, then builds a **web presentation**: slides as an HTML page with styles,
transitions, and optional images or background music. While the deck is being created, the user sees a **live preview**
(slide by slide, as on a screen). When ready, Snappy publishes the page and gives a **link**—viewers open it in the
browser and move through slides with keyboard, touch, or on-screen controls, without installing anything.

For pitches, lessons, reports, and internal updates when a static PDF or a separate slide app is not enough, and the
author wants one link to share.

## 👤 How the user experiences it

1. The user starts the skill or asks for an online presentation as a web page.
2. The agent asks step by step: topic, audience, how many slides, tone, colors, transition style, whether each slide is
   mostly text, image, or mixed, and whether to add photos, logo, or music.
3. The user uploads materials or skips them. The agent uses only what the user provided—no invented facts or figures.
4. The agent shows a short plan (slide titles and purpose); the user confirms or edits the outline.
5. During generation, the chat runs alongside a **live preview** that updates as slides appear; the user can flip through
   finished slides in the preview and ask for tweaks (“simpler slide 3”, “stronger contrast”, “add a chart placeholder”).
6. When satisfied, the user publishes and receives a **shareable link**.
7. The viewer opens the link on a phone, tablet, or desktop: full-screen or comfortable slide view, clear navigation,
   readable text, optional speaker-notes mode for the author if enabled.
