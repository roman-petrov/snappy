# 🧩 Design-system component layouts

## 💡 Concept

**Agent skill.** From a brief and the user’s design tokens (spacing, corners, type, colors), proposes how core UI pieces
should look—buttons, fields, cards, modals, navigation—in a way that fits one system, not random one-off screens.

## 👤 How the user experiences it

1. The user selects which components they need and confirms or pastes their token set.
2. The agent shows one or two layout variants per component with spacing and state notes (normal, hover, error).
3. The user picks variants and receives a clear **spec** for developers or designers.
4. Optional simple mock images help stakeholders agree before build.
