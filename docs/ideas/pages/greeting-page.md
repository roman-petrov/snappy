# 🎊 Interactive greeting page

## 💡 Concept

**Agent skill.** The agent interviews the user, then builds a personal celebration as a web page: text, colors,
animations, photos, optional music. While the page is being created, the user sees it grow in a **phone-sized live
preview**. When ready, Snappy publishes the page and gives a **link** to send—recipients open it in the browser, no app
needed.

For birthdays, weddings, holidays, graduations, team thanks, and other moments when a plain message or a single picture
is not enough.

## 👤 How the user experiences it

1. The user starts the skill or asks for an animated greeting page.
2. The agent asks step by step: occasion, who it is for, tone, colors, animation style, layout (story scroll, one big
   screen, gallery, and so on), what text to show, whether to add photos or music.
3. The user uploads pictures or a track, or skips media. The agent only uses facts the user provided—no invented dates
   or achievements.
4. The agent shows a short summary of the plan; the user confirms.
5. During generation, the chat runs alongside a **mobile preview** that updates as the page appears. The user can ask
   for tweaks (“more confetti”, “softer colors”, “shorter text”).
6. When satisfied, the user publishes and receives a **shareable link**.
7. The recipient opens the link on a phone or desktop: animations, readable text, optional “play music” button—
   comfortable on small screens.
