# Промпт: тестовый markdown-ответ со стримом

Ты генерируешь **один связный markdown-документ** для проверки стриминга. Действуй строго по шагам ниже.

## Шаг 1. Опрос пользователя

Сначала опроси пользователя инструментом `ask`. В каждом поле дай **3–4 варианта**, придумай их сам:

1. О чём будет **таблица**.
2. О чём будут **списки**.
3. Какой **язык программирования** в блоке кода.
4. Что показать в **блоке кода**.

Не дублируй опрос текстом в чате вместо `ask`. **Дождись ответов** — таблицу, списки и код не генерируй до выбора.

## Шаг 2. Общие правила текста

На всём документе соблюдай:

- ~**50 символов** в предложении; абзацы по смыслу — **3–4 предложения**, где уместно.
- Разметка: **жирный**, _курсив_, `обратные кавычки`.
- Только заголовки `#` и `##`; **без** `###` и глубже.
- Не повторяй в абзаце заголовок раздела, если он уже в `##`.
- Без служебных фраз («ниже таблица», «далее список») — только смысл для читателя.

## Шаг 3. Собери ответ по порядку

После выбора пользователя выведи **один ответ подряд** — блоки **в этом порядке**:

### 3.1. Заголовок документа

- Один `#` в самом начале — тема всего ответа.

### 3.2. Введение

- `##` с названием вводного раздела.
- Абзац (2–3 предложения): о чём ответ в целом.
- Абзац с разметкой (**жирный**, _курсив_, `кавычки`).

### 3.3. Таблица

- `##` с названием раздела.
- Абзац (2–3 предложения): зачем таблица и что сравнивается.
- Таблица: **7 строк**, **4 колонки**; ~50 символов в ячейке; тема из выбора пользователя.
- В **5–6 ячейках** — жирный, курсив или обратные кавычки.

### 3.4. Переход к спискам

- `##` с названием связующего раздела.
- Абзац (2–3 предложения): как таблица связана со списками.
- Абзац с разметкой.

### 3.5. Списки

- `##` с названием раздела.
- Абзац (2–3 предложения): что перечисляется и как устроена иерархия.
- Вложенные списки по теме из выбора:
  - **3 пункта** на 1-м уровне;
  - по **3** на 2-м;
  - по **2** на 3-м;
  - ~50 символов в пункте;
  - чередуй нумерованные и маркированные; вложенный уровень может отличаться от родителя;
  - **3 пункта** списка задач;
  - в пунктах — разметка.

### 3.6. Переход к коду

- `##` с названием связующего раздела.
- Абзац (2–3 предложения): зачем после списков идёт пример кода.
- Абзац с разметкой.

### 3.7. Код

- `##` с названием раздела.
- Абзац (2–3 предложения): что делает пример и на каком языке.
- **Один** блок кода: язык и задача из выбора пользователя; объём по смыслу задачи.

### 3.8. Заключение

- `##` с названием заключительного раздела.
- Абзац (2–3 предложения): итог и главная мысль.
- Абзац с разметкой.
