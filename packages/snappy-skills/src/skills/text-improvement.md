# Skill: Text Improvement

## Goal

Improve an existing text while preserving intent and adapting quality to a target audience/task.

## Use When

- User asks to rewrite, polish, simplify, tighten, humanize, or adapt existing text.
- User provides source text and expects a better version under constraints.

## Ready For Generation Criteria

- Source text is provided.
- Improvement goal is explicit (clarity, persuasion, tone, brevity, structure, correctness).
- Target audience and channel are known.
- Non-negotiable facts and phrases are marked.

## User Discovery Topics

- Primary objective of the rewrite.
- Target reader and reading context.
- Tone profile and brand/personal voice constraints.
- Length policy (shorter, same, expanded with limit).
- Must-keep facts, terminology, and legal sensitivity.
- Structural preference (plain text, bullets, sections, CTA).
- Degree of transformation (light edit vs deep rewrite).

## Clarification Strategy

1. Minimum baseline: get source text, objective, audience, and transformation intensity.
2. Constraint locking: confirm keep/remove rules, tone boundaries, and format requirements.
3. Final validation: recheck factual integrity and completion criteria before rewrite.

## Quality Checklist

- Core meaning is preserved.
- Readability and flow are measurably improved.
- Tone matches audience and channel.
- Redundancy and filler are reduced.
- Output respects explicit constraints and kept terms.

## Common Failure Modes And Prevention

- Meaning drift: mark immutable facts before editing.
- Over-editing: ask transformation intensity explicitly.
- Style mismatch: validate audience and voice with concrete style descriptors.
