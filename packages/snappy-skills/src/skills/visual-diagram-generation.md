# Skill: Visual Diagram Generation

## Goal

Generate a clear visual diagram specification and prompt that communicates structure, flow, and hierarchy correctly.

## Use When

- User asks for process, architecture, data flow, org, mind-map, or system diagrams.
- User needs explanatory visual that prioritizes readability over decoration.

## Ready For Generation Criteria

- Diagram purpose and audience are identified.
- Diagram type is selected.
- Entities/nodes and relationships are defined.
- Complexity level and scope boundary are fixed.

## User Discovery Topics

- Communication goal (explain, compare, decide, onboard, document).
- Audience technical level and expected detail.
- Diagram type and notation preferences.
- Mandatory nodes, edges, and grouping logic.
- Directionality and sequencing requirements.
- Label language, naming constraints, and terminology consistency.
- Output format context (presentation slide, document, social post, print).

## Clarification Strategy

1. Minimum baseline: capture objective, audience, diagram type, and core entities.
2. Structural precision: validate relationships, direction, hierarchy, and label semantics.
3. Final validation: confirm completeness, simplicity target, and forbidden ambiguities.

## Quality Checklist

- Diagram tells one coherent story.
- Node and edge semantics are unambiguous.
- Visual hierarchy guides reading order.
- Labeling is concise and consistent.
- Scope is controlled to avoid cognitive overload.

## Common Failure Modes And Prevention

- Overly complex graph: enforce scope cap and optional phased diagrams.
- Ambiguous relations: require explicit edge meaning and direction.
- Visual noise: prioritize clarity, spacing, and hierarchy over stylistic effects.
